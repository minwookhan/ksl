* 실행 과정
#+begin_example
1 # 가상 환경 활성화 (아직 안 했다면)
2 # source .venv/bin/activate
3
4 # main.py 실행 예시:
5 python src/main.py \
6     /path/to/your/video.mp4 \
7     "100,100,640,480" \
8     --server "localhost:50051" \
9     --model-path protos/pose_landmarker_full.task \
#+end_example

10 --output output/packets.bin\\
11 --log-level DEBUG

각 인자에 대한 설명:

- =python src/main.py=: src/main.py 스크립트를 실행합니다.
- =/path/to/your/video.mp4=:
  - 필수: 분석할 비디오 파일의 실제 경로입니다.
  - 예: /home/user/videos/my_sign_language.mp4
- ="100,100,640,480"=:
  - 필수: ROI(Region of Interest)를 지정하는 문자열입니다. x,y,w,h 형식으로, 각 값은 정수여야 합니다.
  - x: ROI 좌측 상단 모서리의 x 좌표
  - y: ROI 좌측 상단 모서리의 y 좌표
  - w: ROI의 너비
  - h: ROI의 높이
- =--server "localhost:50051"=:
  - 옵션 (기본값: =localhost:50051=): gRPC 서버의 주소와 포트입니다.
  - 예: --server "192.168.0.154:50051"
- =--model-path protos/pose_landmarker_full.task=:
  - 옵션 (기본값: =protos/pose_landmarker_full.task=): MediaPipe Pose Landmarker 모델 파일(*.task)의 경로입니다.
  - 기본값이 현재 ksl-cli 디렉토리 기준 상대 경로로 설정되어 있습니다.
- =--output output/packets.bin=:
  - 옵션: 처리된 Protobuf 패킷을 바이너리 파일(*.bin)로 저장할 경로입니다. 이 옵션을 지정하면 gRPC 전송과 별개로 로컬 파일로 저장됩니다.
- =--log-level DEBUG=:
  - 옵션 (기본값: =INFO=): 로깅 레벨을 설정합니다. DEBUG, INFO, WARNING, ERROR, CRITICAL 중 하나를 선택할 수 있습니다. DEBUG를 선택하면 더 자세한 내부 동작 로그를 볼 수 있습니다.

실제 실행 전 확인 사항:

1. 가상 환경 활성화: 반드시 source .venv/bin/activate 명령으로 가상 환경을 활성화한 후 실행해야 합니다.
2. =ksl-cli= 디렉토리에서 실행: 위 예시 명령어는 ksl-cli 디렉토리 안에 있을 때 작동하도록 경로가 설정되어 있습니다.
3. 모델 파일 존재: --model-path로 지정된 .task 파일이 실제로 해당 경로에 존재하는지 확인해야 합니다. (이전 단계에서 protos/pose_landmarker_full.task를 다운로드하여 저장했습니다.)
4. 비디오 파일 존재: /path/to/your/video.mp4에 실제 비디오 파일이 있어야 합니다.
5. gRPC 서버 실행 여부: --server로 지정된 gRPC 서버가 실행 중이어야 실제 네트워크 통신이 가능합니다. (테스트 환경에서는 Mocking으로 우회했지만, 실제 실행에서는 서버가 필요합니다.)
